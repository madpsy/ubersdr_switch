REST API

All endpoints return JSON. Mutating calls accept any method (canonical: POST). $H = this device (e.g. its IP or ESP-XXXXXX.local).

Endpoints

MethodPathPurpose
GET/api/statuscurrent state
POST/api/antenna?position=Nselect port N (0=GROUND)
POST/api/antenna/upstep +1
POST/api/antenna/downstep −1
POST/api/antenna/groundselect GROUND
GET/POST/api/max?value=Nget / set max (0–7)
POST/api/max/bumpbump max (wraps)
GET/api/namesall custom names
POST/api/name?position=N&name=…name ANT N (blank clears)
DELETE/api/name?position=Nclear ANT N name
GET/api/infodevice / network / firmware info
GET/api/mqttread MQTT config + connection status
POST/api/mqttupdate MQTT config (partial JSON OK)
GET/api/ntpread NTP config + sync status + timezone
POST/api/ntpupdate NTP config (host, port, tz_offset)
GET/api/eventslast 25 antenna-change events (JSON array)
POST/api/displayshow custom OLED message {"text":"…","duration":N,"align":"center"}
GET/POST/api/display/blankget / set screen-saver config {"enabled":bool,"timeout":N}
GET/POST/api/display/modeget / set OLED mode {"mode":"port"} or {"mode":"clock"}
GET/POST/api/default-portget / set startup default port {"default_port":N} (0=GROUND, 1–7=ANT)
GET/POST/api/startup-portget / set startup port behaviour {"mode":"default"} or {"mode":"last"}
GET/api/lockread lock state {"locked":bool}
POST/api/lockset lock state {"locked":true|false}
POST/api/lock/toggletoggle lock state
GET/api/scheduleread all schedule entries + global flags
POST/api/scheduleset entry {id,enabled,days,hour,minute,position} or global flags {scheduler_enabled:bool} / {respect_lock:bool}
DELETE/api/schedule?id=Nclear/disable slot N (0–9)
POST/api/restartreboot the device

curl examples

# read status
curl "http://this-device/api/status"

# go straight to ANT3
curl -X POST "http://this-device/api/antenna?position=3"

# go to GROUND
curl -X POST "http://this-device/api/antenna/ground"

# step up / down
curl -X POST "http://this-device/api/antenna/up"
curl -X POST "http://this-device/api/antenna/down"

# set max selectable to 5
curl -X POST "http://this-device/api/max?value=5"

# name ANT3 (blank name clears it)
curl -X POST "http://this-device/api/name?position=3&name=80m%20Vertical"
curl -X DELETE "http://this-device/api/name?position=3"

# OLED display mode (port=antenna prominent, clock=time prominent)
curl "http://this-device/api/display/mode"
curl -X POST -H "Content-Type: application/json" \
     -d '{"mode":"clock"}' "http://this-device/api/display/mode"

# schedule: read all entries
curl "http://this-device/api/schedule"

# schedule: add/update slot 0 — ANT3 at 08:00 Mon–Fri (days bitmask: Mon=2,Tue=4,Wed=8,Thu=16,Fri=32 → 62)
curl -X POST -H "Content-Type: application/json" \
     -d '{"id":0,"enabled":true,"days":62,"hour":8,"minute":0,"position":3}' \
     "http://this-device/api/schedule"

# schedule: every day at 22:00 switch to GROUND (position 0)
curl -X POST -H "Content-Type: application/json" \
     -d '{"id":1,"enabled":true,"days":127,"hour":22,"minute":0,"position":0}' \
     "http://this-device/api/schedule"

# schedule: disable slot 0
curl -X POST -H "Content-Type: application/json" \
     -d '{"id":0,"enabled":false}' "http://this-device/api/schedule"

# schedule: delete slot 0
curl -X DELETE "http://this-device/api/schedule?id=0"

# schedule: master on/off (pauses all entries without deleting them)
curl -X POST -H "Content-Type: application/json" \
     -d '{"scheduler_enabled":false}' "http://this-device/api/schedule"

# schedule: set respect-lock flag
curl -X POST -H "Content-Type: application/json" \
     -d '{"respect_lock":true}' "http://this-device/api/schedule"

# device info + reboot
curl "http://this-device/api/info"
curl -X POST "http://this-device/api/restart"

# MQTT config
curl "http://this-device/api/mqtt"
curl -X POST -H "Content-Type: application/json" \
     -d '{"enabled":true,"host":"192.168.1.10","retain":true}' \
     "http://this-device/api/mqtt"

# rename the switch (OLED splash + web UI header)
curl -X POST -H "Content-Type: application/json" \
     -d '{"device_name":"ShackSDR"}' "http://this-device/api/mqtt"

# NTP: check sync status / set server / set timezone (minutes east of UTC)
curl "http://this-device/api/ntp"
curl -X POST -H "Content-Type: application/json" \
     -d '{"host":"pool.ntp.org","port":123}' "http://this-device/api/ntp"
curl -X POST -H "Content-Type: application/json" \
     -d '{"tz_offset":60}' "http://this-device/api/ntp"

# event log (last 25 changes, newest first)
curl "http://this-device/api/events"

# custom OLED message — multi-line with \n, optional duration & align
curl -X POST -H "Content-Type: application/json" \
     -d '{"text":"ANT 3\n80m Vert","duration":4,"align":"center"}' \
     "http://this-device/api/display"

# JSON body form
curl -X POST -H "Content-Type: application/json" \
     -d '{"position":2}' "http://this-device/api/antenna"

MQTT commands (requires commands: true in Settings → MQTT)

Default prefix: uberant/<hostname>. Override in Settings.

TopicPayloadEffect
<prefix>/set0–7 | up | down | groundSelect antenna / step
<prefix>/max/set0–7Set max antennas
<prefix>/name/set{"position":N,"name":"…"}Set/clear a port name
<prefix>/display{"text":"…","duration":N,"align":"…"}Show custom OLED message
<prefix>/lock"lock" / "unlock" / "toggle"Lock or unlock antenna selection
<prefix>/restartanyReboot the device
# mosquitto_pub examples
mosquitto_pub -h … -t "uberant/ESP-XXXXXX/set" -m "3"
mosquitto_pub -h … -t "uberant/ESP-XXXXXX/set" -m "up"
mosquitto_pub -h … -t "uberant/ESP-XXXXXX/set" -m "ground"
mosquitto_pub -h … -t "uberant/ESP-XXXXXX/max/set" -m "5"
mosquitto_pub -h … -t "uberant/ESP-XXXXXX/name/set" \
  -m '{"position":3,"name":"80m Vertical"}'
mosquitto_pub -h … -t "uberant/ESP-XXXXXX/display" \
  -m '{"text":"ANT 3\n80m Vert","duration":4,"align":"center"}'

Factory reset

Erase all stored settings and WiFi credentials. The device will reboot into the WiFi setup portal. This cannot be undone.

⚠ Erase all settings?

This will permanently erase:

  • WiFi credentials (device will enter setup portal)
  • All antenna names
  • Max antennas setting
  • MQTT configuration
  • NTP / timezone settings
  • Display mode & blank settings
  • Default port & startup port mode
  • All schedule entries

This cannot be undone. The device will reboot immediately.
