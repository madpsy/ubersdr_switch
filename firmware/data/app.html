REST API reference

All endpoints return JSON. Mutating calls accept any method (canonical: POST). $H = this device (e.g. its IP or ESP-XXXXXX.local).

Endpoints

MethodPathPurpose
GET/api/statuscurrent state
POST/api/antenna?position=Nselect port N (0=GROUND)
POST/api/antenna/upstep +1
POST/api/antenna/downstep −1
POST/api/antenna/groundselect GROUND
GET/POST/api/max?value=Nget / set max (0–7)
POST/api/max/bumpbump max (wraps)
GET/api/namesall custom names
POST/api/name?position=N&name=…name ANT N (blank clears)
DELETE/api/name?position=Nclear ANT N name
GET/api/infodevice / network / firmware info
GET/api/mqttread MQTT config + connection status
POST/api/mqttupdate MQTT config (partial JSON OK)
GET/api/ntpread NTP config + sync status
POST/api/ntpupdate NTP config (host, port)
GET/api/eventslast 25 antenna-change events (JSON array)
POST/api/restartreboot the device

curl examples

# read status
curl "http://this-device/api/status"

# go straight to ANT3
curl -X POST "http://this-device/api/antenna?position=3"

# go to GROUND
curl -X POST "http://this-device/api/antenna/ground"

# step up / down
curl -X POST "http://this-device/api/antenna/up"
curl -X POST "http://this-device/api/antenna/down"

# set max selectable to 5
curl -X POST "http://this-device/api/max?value=5"

# name ANT3 (blank name clears it)
curl -X POST "http://this-device/api/name?position=3&name=80m%20Vertical"
curl -X DELETE "http://this-device/api/name?position=3"

# device info + reboot
curl "http://this-device/api/info"
curl -X POST "http://this-device/api/restart"

# MQTT config
curl "http://this-device/api/mqtt"
curl -X POST -H "Content-Type: application/json" \
     -d '{"enabled":true,"host":"192.168.1.10","retain":true}' \
     "http://this-device/api/mqtt"

# rename the switch (OLED splash + web UI header)
curl -X POST -H "Content-Type: application/json" \
     -d '{"device_name":"ShackSDR"}' "http://this-device/api/mqtt"

# NTP: check sync status / set server
curl "http://this-device/api/ntp"
curl -X POST -H "Content-Type: application/json" \
     -d '{"host":"pool.ntp.org","port":123}' "http://this-device/api/ntp"

# event log (last 25 changes, newest first)
curl "http://this-device/api/events"

# JSON body form
curl -X POST -H "Content-Type: application/json" \
     -d '{"position":2}' "http://this-device/api/antenna"

MQTT commands (requires commands: true in Settings → MQTT)

Default prefix: ubersdr/<hostname>. Override in Settings.

TopicPayloadEffect
<prefix>/set0–7 | up | down | groundSelect antenna / step
<prefix>/max/set0–7Set max antennas
<prefix>/name/set{"position":N,"name":"…"}Set/clear a port name
<prefix>/restartanyReboot the device
# mosquitto_pub examples
mosquitto_pub -h … -t "ubersdr/ESP-XXXXXX/set" -m "3"
mosquitto_pub -h … -t "ubersdr/ESP-XXXXXX/set" -m "up"
mosquitto_pub -h … -t "ubersdr/ESP-XXXXXX/set" -m "ground"
mosquitto_pub -h … -t "ubersdr/ESP-XXXXXX/max/set" -m "5"
mosquitto_pub -h … -t "ubersdr/ESP-XXXXXX/name/set" \
  -m '{"position":3,"name":"80m Vertical"}'
